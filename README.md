# AI Agent 软件工程哲学

[English Version](README_en.md) | **中文文档**

本仓库致力于探索和实现 Multi SWE Agent 时代下的软件工程基础哲学。

## 核心理念

本技能基于以下理念开发：

> 在多 Agent 软件工程（Multi SWE Agent）时代，模型能力固定的前提下，"Token 无限"意味着"人力资源无限"。主导所有设计模式的第一原理，应是"让数量庞大、各有所长的 Agent 协同高效地编写代码"，而非"依赖单个天才 Agent 无序地独立开发"。要实现"让数量庞大的 Agent 协同合作"，就必须严格遵循人类已有的全部软件工程管理方式：
>
> 1.  **严格遵照自顶向下设计 (Top-Down Design)**：先写规格（Spec），再选技术栈（Tech Stack），再完成设计（Design），再设计模块（Module），再分配完成编码（Coding）。
> 2.  **代码即"一次性产物"，用完就扔 (Disposable Code)**：如果一份代码库反复修改，最终必将成为"技术债务高企的难以维护代码"，大量迭代积累后，根本无法维护；如果把代码视为"一次性产物"，每次修改都从更新规格（spec）开始重新编写，固定接口（endpoint）和功能，每次重写、每次重写、每次重写，就不存在难以维护的旧代码。
> 3.  **用大量、巨量、海量的单元测试 (Unit Test) 来保证稳定性**：用大量、巨量、海量的单元测试来保证整个代码库的稳定性和 API、endpoint 等等的一致性。既然拥有了大量通用型程序员（Agent），就应再配备大量专业的测试开发人员（SDE Test Agent）来专门完成测试工作。
>
> 人类对于软件开发的所有经验，都在软件工程这门课里。这门课告诉我们唯一一件事，就是如何高效地管理、引导和安排这些通用型 Agent 的工作。在 Token 无限 = 人力无限的今天，软件工程就是最好的管理学圣经。只有彻底吃透了软件工程的精髓，才能让你从"依赖单个天才 LLM Agent 编写代码"，转变到"正确管理1000个会犯错且持续学习的 LLM Agent 协作编写代码"的思路。

## 技能包结构

本仓库包含一个完整的 AI Agent 技能包，位于根目录：

- `SKILL.md` - 技能本体，包含完整的软件工程哲学内容
- `.clawdhub/origin.json` - 技能元数据，用于 Clawdhub 注册表识别

这个技能包可以被其他 AI 代理或系统识别和加载，作为 Multi SWE Agent 开发的指导哲学。

## 使用方式

作为技能加载时，AI Agent 将学习以下核心理念：

1. **Top-Down Design** - 自顶向下的系统设计方法
2. **Disposable Code** - 将代码视为一次性产物的开发理念
3. **Massive Unit Testing** - 用海量测试保证稳定性
