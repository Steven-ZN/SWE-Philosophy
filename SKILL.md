# AI Agent 软件工程哲学：人力无限时代的设计模式

## 核心理念：Token无限 = 人力无限

在 Multi SWE Agent 时代，模型能力固定的前提下，Token 无限等同于人力无限。这意味着我们不再受限于单个"天才"LLM Agent 的能力，而是可以通过协作来解决复杂问题。

## 设计模式主导：让1000个基础 Agent 协作写代码

我们的首要设计原则是"让1000个基础 Agent（会犯错且持续学习的 LLM Agent）协作写代码"，而不是"让一个天才（LLM Agent）随意编写"。为了实现高效的协作，必须严格遵照人类已有的全部软件工程管理方式：

### 1. 严格遵照 Top-Down Design (自顶向下设计)

-   **先写 Spec (规格)**：明确需求和功能定义。
-   **再选 Tech Stack (技术栈)**：根据需求选择合适的技术工具和框架。
-   **再完成 Design (设计)**：进行详细的系统和架构设计。
-   **再设计 Module (模块)**：将系统拆分为独立的模块。
-   **再分配完成 Coding (编码)**：将编码任务分配给各个代理。

### 2. 代码即一次性资源，用完即弃 (Disposable Code)

-   如果一份 codebase 反复修改，最终一定会形成技术债务，大量 sprint 积累后，根本无法维护。
-   如果把代码当一次性资源，每次修改都重写一遍，每次修改需求都从 spec 修改，固定 endpoint 和功能，每次重写、每次重写、每次重写，就不存在技术债务。

### 3. 用大量、巨大量、海量的 Unit Test (单元测试) 来保证稳定性

-   用大量、巨大量、海量的单元测试来保证整个 codebase 的稳定性和 API、endpoint 等等的一致性。
-   既然你有了 1000 个基础程序员，就再雇 1000 个 SDE Test 来专门完成测试工作。

## 总结：Software Engineering 是管理学圣经

人类对于软件开发的所有经验，都在 software engineering 这一门课里。

这一门课告诉我们唯一的一件事，就是如何管理好团队，如何用好团队，如何给团队安排好工作。

在 Token 无限 = 人力无限的今天，software engineering 就是最好的管理学圣经。只有彻底吃透了 software engineering 的精髓，才能让你从"用一个天才 LLM Agent 写代码"，到"正确管理1000个会犯错且不停犯错的 LLM Agent 写代码"的思路转变。
